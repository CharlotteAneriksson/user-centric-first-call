Differens between text-editor, code-editor and IDE

  1. Texteditor simply writes text, ex. notepad. No help, not any built in stuff that colors your code or no code snippets, no nothing. You can use it to learn, absolutly but i would recommend a code editor.
  2. Code editor is designed special to write code for programs. It colors your code nicely which makes it more readable. This type helps you to focus and really learn the code, not any advanced stuff like autocomplete which acually can make your learning process longer since your not typing the code yourself.
  3. Integrated Development Environment, IDE i a more advanced editor that will save you alot of time while writing code. It has extensions like codesnippets, live-server, autocomplete and alot more. This might take the focus of the codeing language away do to the fact theres alot to learn in the IDE itself.

Visual Studio Code

IDE

Open source IDE created by Microsoft for windows, linus and Mac.
Supports alot of different languages, debugging, embedded Git and GitHub, syntax highlights and great code completion.
Allow user to change themes, make your own settings for keyboard shortcuts and allow you to download a bunch of extensions.

In may 2019 they released the online version, I havent tried it yet but i would like to.

Sublime

Code-editor

Free software licensed code editor with a Python API. Supports alot of languages too and colors your code real nice. Makes it easier to learn and understand the syntax.
If you need any functions you need to use plugins. Great to start with, great offline editor and its super easy to set up.

Atom

Code-editor and IDE

This editor has both an offline and an online IDE. They have plugins written in Node.js and also embedded Git Control.
Never tried but I hear it is awesome.

Gitpod

Online IDE

Free open source but only up to 100h /month which acually is pretty much depending on how they count the time.

Just heard about this one, i know a few students use it. As GitHub say themself, One-click ready-to-code development enviroment for github. Saves alot of time with deploying and pushing to github apparently.
